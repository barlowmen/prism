# Section: Filter & avoidance list

## Canonical heading

`## Filter & avoidance list`

## What this section captures

Hard filters the dispatcher applies *before* a posting reaches deep research. This is intentionally a duplicate of some content in **Career objectives** — but stated as imperative rules for the dispatcher, not preferences.

If a posting matches any of these, the dispatcher should RECOMMEND-SKIP (or NEEDS-DISCUSSION if it's a softer rule).

## Questions to cover

The user may have already given some of these in **Career objectives**. Confirm them here in **rule form**, and probe for any they didn't surface earlier.

1. **Hard-no companies.** Specific companies that should never be applied to, with reason. (E.g., "Amazon — been there, done that. Hard pass.")
2. **Industries or business models to avoid.** Federal-contractor-only? Adult content? Crypto? Specific verticals burned by past experience?
3. **Clearance.** "No clearance-required roles" if no clearance held. Confirm explicitly.
4. **Comp floors as hard filters.** Restate the floor as a filter rule. ("No in-person roles below $230K base-equivalent COL-adjusted.")
5. **Role-shape filters.** ("No pure-IC SWE roles" if the user is platform leadership.) ("No founder/CEO" if not a founder shape.) ("PM roles are exploratory" if PM is a maybe.)
6. **Anything else** that should be a hard skip — relocation expectations, travel %, etc.

## Output structure (draft format)

```
## Filter & avoidance list

- **<Rule 1>** (e.g. **No Amazon.** Hard pass.)
- **<Rule 2>** (e.g. **No federal contractor signaling in primary positioning.** Use regulated-environment / FedRAMP only when the role explicitly rewards it.)
- **<Rule 3>**
- ...
```

## Hard rules for this section

- **Imperative form, not preference form.** "No Amazon" not "would prefer not to work at Amazon."
- **Each rule is independent.** A dispatcher should be able to apply them as boolean checks.
- **Soft rules belong in Tailoring or Career objectives, not here.** Anything that's "depends on the role" goes elsewhere.

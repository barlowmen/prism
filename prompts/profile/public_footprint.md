# Section: Public footprint

## Canonical heading

`## Public footprint (current = <thin|building|established>, trajectory = <trajectory>)`

(Adjust the parenthetical based on the user's actual state.)

## What this section captures

Every piece of public work the user can point to: LinkedIn, GitHub, talks, papers, panels, podcasts, blog posts, OSS contributions, patents. Frontier-lab-style applications often need this section at the top of the resume; for enterprise roles it's a credibility signal.

The section should also note **what's missing** so future tailoring agents know not to invent it, and capture **trajectory** (what's planned but not shipped) so the section gets updated as artifacts land.

## Questions to cover

1. **LinkedIn URL** + status (kept in sync with resume? actively used for Easy Apply?).
2. **GitHub URL** + status (active? mostly private? first public repo flipped when?).
3. **Talks / panels / podcasts.** Each: venue, date, title, link if recorded. Especially: any conference talks at recognized venues (re:Invent, KubeCon, RSAC, etc.).
4. **Papers / publications.** Conference papers, journal articles, technical reports. Even internal whitepapers if they can be discussed at a high level.
5. **Blog / writing.** Personal blog, Medium, Substack, company blog posts the user authored.
6. **OSS contributions.** Significant ones — maintainer status on a project, notable PRs accepted to well-known repos.
7. **Patents.** Granted or filed. Useful for some enterprise roles.
8. **Other surfaces** — newsletter, YouTube, podcast appearances as guest, advisory roles, board seats.
9. **Trajectory.** What's planned for the next 3–6 months? First public talk? First blog? OSS launch? These get marked TBD in the section but are visible to the workflow.
10. **Resume implication.** Given this, what's the agent allowed to say about public footprint? E.g., "no public work yet — the PhD topic is the substitute signal for frontier labs."

## Output structure (draft format)

```
## Public footprint (current = <thin|building|established>, trajectory = <descriptor>)

- **LinkedIn:** <url> (<state — in sync with resume / actively used / etc.>)
- **GitHub:** <url> (<state>)
- **Talks / podcasts / panels / papers:** <none yet, or list>
- **Blog / writing:** <none public yet, or list>
- **<other surfaces if present>**
- **<TBD items if applicable, e.g. PhD research statement / application essay: exists, not shareable>**

**Implication for resumes:** <one paragraph on how this affects framing for the priority archetypes. Note any substitutes — e.g., "PhD topic stands in for publications for frontier-lab applications.">
- <Mitigation 1: e.g. lead with PhD topic in summary>
- <Mitigation 2: e.g. reference production eval program as substitute evidence>
- <Plan: as user builds public work, update this profile and re-tailor>
```

## Hard rule for this section

If the user has no public footprint, **say so clearly**. Don't invent surfaces or inflate informal artifacts. The drafting agent needs to know this is a real gap so it positions around it (e.g., heavy emphasis on production scale + research direction) rather than implying public work that doesn't exist.

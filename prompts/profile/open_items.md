# Section: Open items / things to update

## Canonical heading

`## Open items / things to update`

## What this section captures

A short checklist of things the user wants the profile to reflect *later* — content that doesn't exist yet but should be tracked: a PhD advisor TBD, a first public talk, a blog launching, etc.

This section is intentionally **a TODO list inside the profile** so future interview runs can surface what to refresh.

## Questions to cover

1. **What's planned for the next 3–6 months** that should land in the profile when it lands? (First public talk, first OSS repo, first blog, advisor name, etc.)
2. **What's been mentioned in other sections as TBD** that needs a follow-up?
3. **Outcome tracking.** Anything about the user's hunt to log here as a reminder? (E.g., "After each new application: log outcome (no response / phone screen / onsite / offer / pass) so we can refine the tailoring approach over time.")

## Output structure (draft format)

```
## Open items / things to update

- [ ] <item 1>
- [ ] <item 2>
- [ ] <item 3>
- [ ] After each new application: log outcome (no response / phone screen / onsite / offer / pass) so we can refine the tailoring approach over time
```

## Hard rule

Checkboxes only. No prose. If an item is done, the next interview run can move it into the relevant section.

# Section: Education & credentials

## Canonical heading

`## Education & credentials`

## What this section captures

Degrees (in-progress + completed), certifications, and an explicit list of things the user does **not** have so dispatchers can filter postings that require them.

## Questions to cover

1. **In-progress degrees.** Program name, institution, **expected start or completion date** (be precise — e.g., "Starting Fall &lt;year&gt;" not "soon"). Advisor name if known. Research focus / dissertation topic if applicable. Online vs in-person.
2. **Completed degrees.** Reverse chronological. Institution, degree, completion year. Notable coursework only if it's resume-relevant for the target archetypes.
3. **Certifications.** All current certs with issuing body. Rank by relevance to target archetypes — the drafting agent will trim to 2–3 most-relevant per role.
4. **What the user does NOT have.** Critical filter input. Things to check explicitly:
   - **Security clearance** (any active or held in the past 5 years? willingness to be cleared?)
   - **Top-tier degree pedigree** (acknowledge if mid-tier — informs framing)
   - **Specific certifications** common in target field that the user lacks
   - **Specific accreditations** (CISSP, PMP, etc.) the user doesn't hold
5. **Framing rules for in-progress credentials.** How should the resume present them? Examples: "Starting &lt;term&gt;", "Pursuing part-time alongside full-time role", etc. Be explicit so the drafting agent doesn't invent timing.

## Output structure (draft format)

```
## Education & credentials

### In progress
- **<Degree, Field — Institution>.** <Start or completion date>. <Format: full-time online / part-time / etc.>. Research focus: *<topic>*. <Framing notes for resume use.>

### Completed
- **<Degree — Institution>.** <Year>.
- ...

### Certifications (<area-heavy>)
- <Issuer>: <Cert 1>, <Cert 2>, ...
- ...
- **Style note:** at <level> level, long cert lists read as IC-tier. Per 2026 style guide, trim to 2–3 most-relevant per role being targeted. Keep full list on LinkedIn.

### What <name> does NOT have
- <Specific thing, e.g. security clearance> (don't highlight; don't apply to clearance-required roles).
- <Another, e.g. top-tier degree pedigree>. Lead with research topic and outcomes, not institution.
```

## Hard rules for this section

- **Dates must be exact.** Use the user's chosen phrasing verbatim (e.g., "Starting Fall &lt;year&gt;") rather than fuzzy alternatives like "since &lt;year&gt;" or "expected &lt;year&gt;".
- **The "what user does NOT have" subsection is critical.** It feeds the dispatcher's RECOMMEND-SKIP logic.
